# 読書記録サイト プロジェクト指針

## コンセプト

読んだ本の内容を「自分が理解し、他者に説明できる形」でまとめる。
単なる読書メモではなく、**体系的に人に伝わる**ことを目指す。

---

## 3層アプローチ：理解のフレームワーク

```
┌─────────────────────────────────────────┐
│  超抽象化：本質・共通点を抽出           │
├─────────────────────────────────────────┤
│  超構造化：情報を整理・体系化           │
├─────────────────────────────────────────┤
│  超具体化：書かれていた具体的内容       │
└─────────────────────────────────────────┘
```

### 1. 超具体化
- 本に書かれていた具体的な内容を正確に記載
- 引用、事例、エピソードを含める
- 著者の言葉をそのまま伝える

### 2. 超構造化
- 具体的な内容を整理・分類
- カテゴリ分け、番号付け、階層化
- 読者が全体像を把握できる構造を作る

### 3. 超抽象化
- 構造化した情報から共通点・本質を抽出
- 「一言で言うと」で結論を先に提示
- 読者が核心を即座に理解できるようにする

## ページ構成への適用

| 層 | ページ上の表現 | 例（修身教授録） |
|---|---|---|
| 抽象 | 「一言で言うと」セクション | 「人間はいかに生きるべきか」を説いた現代の論語 |
| 構造 | 教えのカテゴリ化・番号付け | 核となる7つの教え |
| 具体 | 各教えの詳細・引用・現代への示唆 | 「人生二度なし」の解説と引用 |

## デザイン哲学

### なぜレイアウトにこだわるか

> AIと違い、人間は見た感じで読む気が変わる

- 内容がどれだけ良くても、見た目で離脱されたら届かない
- レイアウトは「読む気にさせる」ための重要な入り口
- 視覚的な構造が、内容の構造を補強する

### レイアウト原則

1. **結論先行**: 最も抽象的な情報（一言で言うと）を最初に
2. **視覚的階層**: 抽象→構造→具体の流れを視覚的に表現
3. **適切な密度**: 長い縦スクロールを避け、2-3列レイアウトを活用
4. **レスポンシブ**: 14-15インチPC と iPhone の両方で最適表示
5. **余白の活用**: 詰め込みすぎず、読みやすさを優先
